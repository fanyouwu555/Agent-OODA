# 用户指南

本文档详细介绍 OODA Agent 的功能和使用方法。

## 功能概述

OODA Agent 是一个基于 OODA 循环策略的智能代理系统，具备以下核心能力：

- **智能对话**: 基于 OODA 循环的理解和响应
- **文件操作**: 读取、写入、编辑文件
- **网络搜索**: 搜索互联网信息
- **代码执行**: 运行代码片段
- **数据分析**: 分析数据并生成报告
- **图像处理**: 处理图像文件
- **PDF 处理**: 提取和操作 PDF 文档

### 会话管理

- **新建会话**: 点击左侧的 "+ 新建会话" 按钮
- **切换会话**: 点击会话列表中的会话名称
- **删除会话**: 右键点击会话，选择删除

### 消息交互

- **发送消息**: 在底部输入框输入内容，按 Enter 或点击发送按钮
- **查看思考过程**: 点击消息中的 "思考过程" 展开查看
- **查看工具调用**: 工具调用会以卡片形式展示

## 权限系统

系统采用三级权限模式：

| 权限级别 | 说明 | 示例 |
|----------|------|------|
| `allow` | 自动允许 | 文件读取、搜索 |
| `ask` | 需要确认 | 文件写入、命令执行 |
| `deny` | 自动拒绝 | 访问系统敏感目录 |

当执行需要确认的操作时，会弹出确认对话框。

## 配置选项

### 模型选择

在设置面板中可以选择不同的 LLM 提供商：

- **Local Ollama**: 本地部署的模型
- **Kimi**: Moonshot AI 提供的模型
- **LongCat**: 其他 OpenAI 兼容服务

### 系统设置

可以调整以下参数：

- **温度 (Temperature)**: 控制输出的创造性 (0-1)
- **最大 Token**: 限制响应长度
- **超时时间**: 请求超时设置

## 高级功能

### 流式输出

系统支持实时流式输出，可以：
- 实时查看思考过程
- 观察工具调用进度
- 逐步显示生成内容

### 记忆系统

系统具备记忆能力：
- **短期记忆**: 当前会话的上下文
- **长期记忆**: 跨会话的事实、偏好、经验

### 性能监控

每次请求都会显示性能指标：

```json
{
  "observeTime": 5,
  "orientTime": 3,
  "decideTime": 8,
  "actTime": 12,
  "totalTime": 28
}
```

## 最佳实践

### 1. 明确任务描述

提供清晰、具体的任务描述：

**好的示例：**
```
读取文件：/home/user/documents/report.txt，从第 10 行开始，读取 50 行
```

**不好的示例：**
```
读取文件
```

### 2. 使用绝对路径

始终使用绝对路径：

**好的示例：**
```
读取文件：/home/user/workspace/project/file.txt
```

**不好的示例：**
```
读取文件：../file.txt
```

### 3. 分步执行复杂任务

对于复杂任务，建议分步骤执行：

1. 先搜索相关信息
2. 分析搜索结果
3. 基于分析执行操作

### 4. 及时确认权限请求

当弹出权限确认对话框时，及时确认或拒绝，以保持工作流程顺畅。

## 故障排除

### 技能执行失败

**问题：** 技能执行失败，提示权限不足

**解决方案：**
- 检查文件路径是否在工作目录内
- 检查技能是否有所需的权限
- 检查输入参数是否符合要求

### 响应缓慢

**问题：** 系统响应缓慢

**解决方案：**
- 减少历史记录大小
- 切换到更快的模型
- 检查网络连接

### 模型连接失败

**问题：** 无法连接到 LLM 服务

**解决方案：**
- 检查 Ollama 服务是否运行
- 验证 API Key 是否正确设置
- 检查网络连接
