## 关键组件说明

### 1. **消息路由** (`packages/opencode/src/server/routes/session.ts:733`)
- 主要端点：`POST /:sessionID/message`
- 处理用户输入，返回流式 JSON 响应

### 2. **SessionPrompt** (`packages/opencode/src/session/prompt.ts`)
- `prompt()`: 入口函数，创建用户消息
- `loop()`: 主循环，协调 LLM 交互和工具调用
- 1961 行代码，是核心逻辑

### 3. **LLM 层** (`packages/opencode/src/session/llm.ts`)
- `stream()`: 准备调用参数，合并 prompts
- 使用 ai-sdk 的 `streamText()` 函数
- 支持工具调用和流式响应

### 4. **工具系统** (`packages/opencode/src/tool/`)
- 每个工具是一个 TS 文件 (read.ts, edit.ts, bash.ts 等)
- 工具配置包含：schema, description, execute 函数
- Agent 权限系统控制哪些工具可用

### 5. **Agent 配置** (`packages/opencode/src/agent/agent.ts`)
- 内置 agents: build, plan, general, explore, compaction, title, summary
- 每个 agent 有不同的：
  - 权限规则 (哪些工具可用)
  - 系统 prompt
  - 模型配置 (temperature, topP 等)

### 6. **流式处理流程**
```
用户输入 → 创建消息 → 加载历史 → 调用 LLM → 
→ 流式返回文本 → (遇到工具调用) → 执行工具 → 
→ 返回结果给 LLM → 继续流式 → 直到完成
```

### 7. **权限系统** (`packages/opencode/src/permission/next.ts`)
- 每个工具调用前都会检查权限
- 规则: allow/deny/ask
- 可配置 pattern 匹配文件路径

这是一个典型的 **ReAct** (Reasoning + Acting) 模式实现，LLM 在思考、行动、观察之间循环，直到给出最终答案。
